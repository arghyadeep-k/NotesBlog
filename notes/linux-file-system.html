Linux File System

  • /bin ‚Üí Binaries. Basic binaries or programs are present here like ls and cat .
  • /sBin ‚Üí System Binaries. Programs used by system admin.
  • /boot ‚Üí Files used by the OS to boot up. Not to be messed with.
  • /cdrom ‚Üí Legacy mounting point for CD ROMs.
  • /dev ‚Üí Everything lives here. Based on UNIX, it was decided everything is a file. Hardwares are present here. For example, disk is present as /dev/sda/ , /dev/sda1/ , /dev/sda2/ . Webcams and other hardwares are also present here.
  • /etc ‚Üí Et cetera. Sytem wide configurations are stored like apt
  • /lib ‚Üí Libraries. Libraries required by the binaries in bin are present here.
  • /mnt & /media ‚Üí Mounted devices are present here like external HDDs & USB drives. Manually mounted things go to /mnt while automatically mounted devices show up under media.
  • /opt ‚Üí Optional folder. Manually installed softwares installed are present here.
  • /proc ‚Üí sudo files that contain information about procedures and system processes. Every process will have a directory with the process id with related files in it like status files.
    • cat /proc/cpuinfo
    • cat /proc/uptime
  • /root ‚Üí root user's home folder. Needs root access to use files stored in here.
  • /run ‚Üí Newly introduced and used differently by different distros. Temp FS file system which means it runs on RAM. Everything in it is gone when the system is rebooted. It is used by processes that start early in the boot procedure to store the runtime information.
  • /snap ‚Üí Used to store snap packages by Ubuntu
  • /srv ‚Üí Service Directory. Service data is stored. For webservers and ftp servers, the files to be accessed externally are placed here which in turn provides better security.
  • /sys ‚Üí System folder. It can be used to interact with the kernel. Similar to run directory. It's not physically written to the disk but created everytime the system boots up.
  • /tmp ‚Üí Temporary. USed to store files temporarily by applications for a session like saving a temporary copy of a word document.
  • /usr ‚Üí User application space. Used to store applications used by the user. Applications stored here are considered non-essential for basic system operations. Programs installed from source code will end up in the /usr/local/ folder.
  • /var ‚Üí Variables. Contains files & directories that are expected to grow in size.
    • /var/crash/ ‚Üí Holds info about crashes.
    • /var/logs/ ‚Üí Holds logs of applications & system.
    • Databases for mail & temporary storage for printer queues.
  • /home ‚Üí Contains folder for each user. Personal files are stored here. Users can only access their own folder unless they are root user. This folder is often installed on a separate partition so that the OS can be reinstalled without users losing their files. User applications also stores files here sometimes.
